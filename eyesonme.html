Data Sovereignty — End-to-End Encryption

Encryption Architecture

Non-negotiable. All data is encrypted at every layer from moment of capture. No plaintext exists outside operator control at any point. Physical seizure, network interception, legal compulsion — all yield only ciphertext.

Layer 1

Storage — At Rest, On Device

Root filesystem encrypted via LUKS2. Key derived from hardware-bound TPM secret + operator passphrase — passphrase never stored on device. Forced reboot without key = unreadable device. All snapshots, clips, logs written to encrypted partition. Temp files via tmpfs only — no cleartext to unencrypted storage. RAM encryption via TRESOR — keys in CPU registers, immune to cold boot attacks.

Layer 2

Transit — Station to VPS

All outbound traffic exclusively over WireGuard VPN. ChaCha20 + Poly1305 + Curve25519 — modern, audited. If tunnel is down: station buffers locally and retries. No fallback to cleartext. Ever. VPS is operator-controlled, hosted outside United States jurisdiction (Iceland, Switzerland, or Netherlands). No US-based cloud providers in data path.

Layer 3

Application Layer — Payload

All snapshots and clips encrypted with operator public key (libsodium secretbox or GPG) before queuing for upload. Encrypted payload transits WireGuard — double-wrapped. VPS stores only encrypted blobs. Full VPS compromise yields no readable media without operator private key. Private key never leaves operator possession. Never stored on station. Never stored on VPS.

Layer 4

Field Collection

Data pulled via ESP32 session written to operator device as encrypted archive (age or GPG). WPA3 encrypted in transit during session. Station encrypts with operator public key at source before transfer — no plaintext sent even over authenticated local session. Operator decrypts only in isolated environment.

Layer 5

Auth Material

TOTP secrets stored in LUKS-encrypted partition. WireGuard private keys stored in LUKS-encrypted partition. ESP32 WPA3 credentials provisioned at deployment, never transmitted in plaintext. All key material wiped from provisioning device after station setup. Operator-triggered remote wipe via authenticated WireGuard command destroys LUKS header — device forensically sterile within seconds.

Seizure Resistance: Encrypted filesystem requires key not present on device. No credentials, keys, or plaintext recoverable. Forced reboot triggers LUKS lockout. Tamper switch triggers immediate secure wipe. Station attempts final encrypted alert before shutdown. Operator retains sole key custody. No escrow. No recovery service. No third-party key holders.

Encryption is a First Amendment right. You are not required to decrypt your own property. The Fifth Amendment protects against compelled password disclosure. Doe v. United States, 487 U.S. 201 (1988).

Setup Sequence

01

Hardware Assembly

Wire: solar panel, MPPT controller, LiFePO4 battery, buck converter (12V to 5V), RPi 5. Mount camera module. Attach 4G LTE HAT. Connect LoRa + ESP32 modules via SPI/UART. Install in IP65 enclosure with cable glands.

02

OS + Hardening

Flash DietPi to industrial microSD or NVMe. Enable LUKS2 full-disk encryption. Configure unattended-upgrades. Disable all unnecessary services. Set up tmpfs for temp writes. Install WireGuard, OpenCV, heartbeat script.

03

Encryption Stack

Generate operator keypair (GPG or age). Deploy public key to station. Configure application-layer encryption for all captures. Set up WireGuard tunnel to offshore VPS. Verify: no plaintext leaves device at any layer.

04

VPS Configuration

Provision VPS outside US jurisdiction (Iceland, Switzerland, Netherlands). Enable LUKS2 on VPS storage. Configure WireGuard endpoint. Set up rsync/SFTP ingestion for encrypted blobs. Deploy heartbeat monitor + alerting.

05

Local Radio + Field Client

Flash ESP32 firmware for WiFi AP gatekeeper. Configure LoRa TOTP authentication. Provision WPA3 credentials. Install field client CLI on operator device: totp-generate, lora-wake, collect, status, provision.

06

Deployment + Legal

Verify legal standing in your jurisdiction (Wisconsin: complete — other states: verify public photography + encryption rights). Deploy to public location with line-of-sight to target facility. Register station in your EYESONME dashboard fork. Publish. Watch back.

Legal summary: Recording in public spaces is protected under the First Amendment. Encryption is protected speech. The Fifth Amendment protects against compelled password disclosure. No physical tampering, no computer system access, no wiretapping. Check state-specific recording and drone laws before deployment.
